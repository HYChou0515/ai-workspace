# Stop-the-line 判準

「停線」是製造業最貴的決策之一 — 直接吃產能,跨班次的決策升級慢。
但**該停沒停**(讓不良品流到下一站、出貨、客戶手上)比停線本身代價大很多倍。
這份 checklist 幫你 / 用戶決定:**現在的證據撐得起停線嗎?**

## 觸發條件(滿足任一即建議停線)

### Tier 1 — 必停(無爭議)

- [ ] **安全事件**(火、爆、化學物洩漏、機台失控傷人)→ 不討論,立即停。
- [ ] **法規 / 認證紅線**(出口管制、客戶資格喪失)→ 停 + 通知 QA + 上報。
- [ ] **顯著 spec 失效**(下游 100% reject 的關鍵特徵壞掉)→ 停。
- [ ] **客戶投訴**已驗證為製程問題且**有再發風險** → 停 + 上報客訴流程。

### Tier 2 — 強烈建議停(看證據強度)

- [ ] **不良率超過 control limit**(SPC out-of-control)且**連續 N 點同向**(N≥3,看製程要求)
- [ ] **關鍵 CTQ(critical-to-quality)量測**漂出規格,且 root cause 還沒確認
- [ ] **AOI / 終測抓到** *系統性* defects(不是隨機散布,是同一位置 / 同一 lot)
- [ ] **跨班次同樣症狀**(排除單一班次的可疑變數)

### Tier 3 — 考慮停 + 觀察(證據還不夠)

- [ ] 單次離群,但不到 OOC
- [ ] 量測來源還沒驗證(可能 Gage R&R 問題)
- [ ] 上游某工序剛變更,影響面還在收斂

## 停之前要先驗證的東西

不要在 Tier 2/3 就拍板。先快速跑這幾項(每項 5–15 分鐘):

1. **量測本身可信嗎?** Gage 重複量同一個樣本,結果穩定嗎?
2. **是真的問題還是抽樣偏差?** 拉小時段 / 班次 / 機台維度看分布。
3. **跟 fishbone 上的某個變更時間吻合嗎?** SOP 改、原料換、機台 PM 後 = 強烈信號。
4. **下游有沒有救?** 如果 rework 比較便宜 / 不會出貨,可以邊產邊查。
5. **多少不良品已經流到下一站?** 涉及 quarantine 規模。

如果上面三項中有兩項以上失守 → 直接升級到 Tier 1 / 2,停。

## 停了之後的動作清單

- [ ] **保留現場證據**:當下機台參數、近期 log、剛產出的樣本(quarantine,標 hold)
- [ ] **記錄時間**:停線時間、班別、機台 ID、誰拍板停的
- [ ] **同時開兩條軌道**:
  - **救火**:rework / quarantine / 替代產線
  - **追因**:fishbone + 5 Whys(用本 template 的另兩個 skill)
- [ ] **30 分鐘內升級**:當班主管 → 部長 → 廠長(視嚴重度)
- [ ] **預估復產時間**:有就講,沒有就講「待 root cause 確認」

## 反模式

- **猶豫太久** 想撐到下班才停 → 不良品繼續流的代價遠超停線本身
- **停了沒同時追因** → 只是把問題冰起來,下一班還是會踩
- **單看一個量測點拍板** → 跨班次 / 跨機台 / 跨時段都該對齊
- **沒記錄就復產** → 之後寫不出 report v1 + 客戶問起來沒佐證

## 輸出

跟用戶說:
- 目前命中哪幾條觸發條件(Tier 1/2/3 具體哪幾項)
- 還需要驗證哪些(列上面那五點裡未完成的)
- 建議:**停 / 觀察 N 分鐘 / 繼續產**,以及為什麼
- 如果建議停:給上面的「停了之後動作清單」當下一步
